六、AI 图像生成方案 ### 6.1 角色提取 + 模板融合 **流程：** 1. 用户上传角色图 2. AI 提取角色特征（人脸、衣着、风格） 3. 将角色特征融合进用户选择的 16 张模板切片 4. 输出 16 张融合图，拼接成 4×4 宫格 ### 6.2 AI API 选型 | 方案 | API | 成本 | 效果 | 速度 | |------|-----|------|------|------| | 推荐 | 通义万相 Wanx2.1 | ~0.1元/张 | ⭐⭐⭐ | 快 | | 备选 | Flux Dev | ~0.05元/张 | ⭐⭐⭐⭐ | 中 | | 备选 | Midjourney | ~0.2元/张 | ⭐⭐⭐⭐⭐ | 慢 | ### 6.3 优化策略 - 首批上线先用**模板融合**方案（简单换脸，不走 AI 生成，大幅降低成本） - 用户量大后升级 AI 生成方案 - 静态生成（16宫格）优先，动态 GIF 后续再上 --- ## 七、开发里程碑 ### Phase 1：基础框架（1-2天） - [ ] 项目初始化（React + Vite + TailwindCSS） - [ ] 页面结构搭建（首页/作品/充值/个人中心） - [ ] 登录流程（微信扫码登录，后端对接） - [ ] 静态模板系统（12个主题，预先画好的素材） ### Phase 2：核心功能（2-3天） - [ ] 图片上传功能 - [ ] 模板选择交互 - [ ] 生成任务流程 - [ ] 任务列表页 - [ ] 能量余额系统 ### Phase 3：付费系统（1-2天） - [ ] 充值套餐页 - [ ] 支付流程（先接易支付/码支付） - [ ] 订单系统 - [ ] 能量到账 ### Phase 4：AI 能力（2-3天） - [ ] 角色提取 API 接入 - [ ] 模板融合/AI 生成 - [ ] 动态 GIF 生成（可选） ### Phase 5：上线优化（1天） - [ ] 域名解析配置 - [ ] 数据库初始化 - [ ] 压力测试 - [ ] 上线 **预计总工期：7-12 天** --- ## 八、待确认事项 1. **域名：** 域名是什么？我需要配置 DNS 解析 2. **数据库：** 数据库连接信息（Host/端口/账号/密码） 3. **AI API：** 打算接入哪个 AI 图像 API？或者先用模板融合方案？ 4. **支付渠道：** 微信支付商户资料是否已有？还是先用第三方聚合支付？ 5. **Logo/名字：** 最终产品叫什么名字？ --- ## 九、参考站点功能清单 | 功能 | 状态 | 备注 | |------|------|------| | 微信扫码登录 | ✅ | 需微信开放平台资质 | | 模板主题选择 | ✅ | 12个主题 | | 切片预览 | ✅ | 点击看大图 | | 角色图上传 | ✅ | 拖拽上传 | | 16宫格生成 | ✅ | 核心功能 | | 动图生成 | ✅ | 额外功能 | | 能量系统 | ✅ | 付费货币 | | 充值套餐 | ✅ | 6档套餐 | | 任务列表 | ✅ | 分页+状态筛选 | | 法律文档同意 | ✅ | 隐私/服务协议 | --- *文档确认后开始开发。如有调整请随时告知。*
